“核心浅浅 / 分身 / 群聊”生命周期说明（公开版）

为避免“随机生成角色”的不透明体验，产品将“核心浅浅”“分身”“群聊”统一为有状态生命周期实体，并按以下状态机执行。

1) 分身创建触发（Create）

  • 当同一主题出现稳定偏好（如 3 次以上被重复强调）；
  • 当用户主动请求“需要一个更理性/更执行导向的角色”；
  • 当长期目标形成独立上下文（例如学习、工作、情绪支持）。

2) 分身稳定参数（Stable Persona Params）

每个分身至少绑定以下稳定参数：

  • role_tone：角色语气（温柔 / 理性 / 直接等）；
  • focus_scope：关注范围（情绪支持 / 目标推进 / 关系复盘）；
  • boundary_rule：边界与禁区；
  • memory_visibility：可读取的记忆范围；
  • activation_level：激活等级（活跃 / 观察 / 冻结）。

3) 群聊创建条件（Group Create）

  • 至少两个分身与同一目标相关；
  • 存在明确协作需求（冲突调和、方案评审、角色分工）；
  • 用户主动确认“创建群聊”。

4) 合并 / 冻结机制（Merge / Freeze）

  • 合并：两分身参数相似度与历史任务重叠持续升高；
  • 冻结：30 天以上未被调用，或用户手动暂停；
  • 解冻：用户主动唤醒，或出现强匹配场景。

该说明用于提高系统可解释性，减少“凭空长出角色”的不确定感。返回主页：齐浅浅。
